# Modernizing My Portfolio with Next.js 15 and TypeScript

After years of using a legacy React template for my portfolio, I decided it was time for a complete overhaul. This post chronicles my journey of rebuilding the entire portfolio using modern web technologies.

## Why the Rebuild?

My old portfolio was built with:
- Create React App (outdated)
- JavaScript (no type safety)
- Basic CSS with some Bootstrap
- Static content hardcoded in components

The problems were clear:
- **Poor Developer Experience**: No TypeScript, limited tooling
- **Maintenance Issues**: Hardcoded content, difficult to update
- **Performance**: Large bundle sizes, no optimization
- **SEO**: Client-side rendering, poor meta tag management
- **Design**: Outdated UI patterns and limited responsiveness

## Technology Stack Selection

After evaluating various options, I settled on this modern stack:

### Frontend Framework: Next.js 15
- **App Router**: Latest routing paradigm with layouts and nested routes
- **Turbopack**: Lightning-fast development builds
- **Static Site Generation**: Perfect for portfolio sites
- **Built-in Optimization**: Image optimization, font optimization, etc.

### Language: TypeScript
- **Type Safety**: Catch errors at compile time
- **Better DX**: Excellent IntelliSense and refactoring
- **Self-Documenting**: Types serve as documentation
- **Team Collaboration**: Easier to maintain and scale

### Styling: Tailwind CSS 3
- **Utility-First**: Rapid development without context switching
- **Design System**: Consistent spacing, colors, and typography
- **Responsive**: Mobile-first approach built-in
- **Dark Mode**: Built-in dark mode support

### Content Management: Contentlayer 2
- **Type-Safe MDX**: Generate TypeScript types from content
- **Git-Based**: Version control for content
- **Local Development**: Fast builds and hot reloading
- **Flexible**: Support for multiple content types

## Architecture Decisions

### Component Architecture
I opted for a component-based architecture with clear separation of concerns:

```typescript
// Example: Button component with variants
interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'default' | 'secondary' | 'outline' | 'ghost';
  size?: 'sm' | 'md' | 'lg' | 'icon';
  asChild?: boolean;
}

export const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, ...props }, ref) => {
    return (
      <button
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        {...props}
      />
    );
  }
);
```

### Design System
Created a comprehensive design system with:
- **Color Tokens**: Semantic color variables for theming
- **Typography Scale**: Consistent text sizing and spacing
- **Component Variants**: Flexible component APIs
- **Animation System**: Smooth transitions and micro-interactions

### Content Strategy
Structured content using Contentlayer with three main types:
- **Pages**: Static content like About, Contact
- **Projects**: Portfolio projects with metadata
- **Blog Posts**: Technical articles and updates

## Lessons Learned

### TypeScript is a Game Changer
The type safety and developer experience improvements are substantial. Catching errors at compile time and having excellent IntelliSense makes development much more pleasant.

### Tailwind CSS Scales Well
Initially skeptical about utility-first CSS, I now appreciate how it scales for larger projects. The consistency it enforces and the speed of development is remarkable.

### Content as Code Works
Using Contentlayer to manage content in Git alongside code is powerful. It enables version control, type safety, and keeps everything in sync.

### Performance by Default
Next.js's built-in optimizations provide excellent performance without additional configuration. The static generation approach is perfect for portfolio sites.
